Полное руководство по созданию сайтов

От основ HTML до публикации вашего первого сайта. Всё, что нужно знать начинающему веб-разработчику.

Основы создания сайтов

Создание сайта может показаться сложной задачей, но на самом деле это процесс, который можно разбить на простые шаги. В этом разделе мы рассмотрим фундаментальные понятия веб-разработки.

Что такое веб-сайт?

Веб-сайт — это набор связанных веб-страниц, доступных через интернет. Каждая страница создается с использованием языков разметки и программирования, которые понимают веб-браузеры.

Сервер

Компьютер, на котором хранятся файлы сайта и который отвечает на запросы пользователей, отправляя им эти файлы.

Клиент

Устройство пользователя (компьютер, телефон) с браузером, который запрашивает страницы с сервера и отображает их.

HTTP/HTTPS

Протоколы передачи данных между сервером и клиентом. HTTPS — защищенная версия HTTP с шифрованием.

Три основные технологии веб-разработки

HTML

Язык разметки, который определяет структуру содержимого веб-страницы. С помощью HTML вы создаете заголовки, абзацы, изображения, ссылки и другие элементы.

Пример: <h1>Заголовок</h1> создает заголовок первого уровня.

CSS

Язык стилей, который описывает внешний вид HTML-элементов. CSS позволяет задавать цвета, шрифты, расположение элементов и создавать адаптивный дизайн.

Пример: h1 { color: blue; } делает все заголовки синими.

JavaScript

Язык программирования, который добавляет интерактивность на веб-страницы. С помощью JS можно создавать анимации, обрабатывать события пользователя и многое другое.

Пример: document.getElementById("demo").innerHTML = "Привет!"; изменяет содержимое элемента.

Совет: Начинайте с простых проектов и постепенно увеличивайте сложность. Не пытайтесь изучить все технологии сразу.

Как браузер отображает сайты?

Когда вы вводите адрес сайта в браузере, происходит следующее:

1

Запрос к DNS-серверу

Браузер преобразует доменное имя (например, google.com) в IP-адрес сервера.

2

Запрос к серверу

Браузер отправляет HTTP-запрос на сервер с этим IP-адресом.

3

Ответ сервера

Сервер отправляет HTML, CSS, JavaScript файлы и другие ресурсы (изображения, шрифты).

4

Парсинг и рендеринг

Браузер анализирует HTML, строит DOM-дерево, применяет CSS стили и выполняет JavaScript код.

5

Отображение страницы

Браузер отображает готовую страницу на экране.

Структура HTML-документа

HTML (HyperText Markup Language) — это язык разметки, который используется для создания веб-страниц. Каждый HTML-документ имеет определенную структуру, которую понимают браузеры.

Минимальная структура HTML-документа

<!DOCTYPE html>
<html lang="ru">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Название страницы</title>
</head>
<body>
    <!-- Содержимое страницы -->
    <h1>Привет, мир!</h1>
    <p>Это моя первая веб-страница.</p>
</body>
</html>

Объяснение ключевых элементов:

<!DOCTYPE html>

Декларация типа документа - сообщает браузеру, что это документ HTML5. Это должна быть первая строка в любом HTML-документе.

<html></html>

Корневой элемент - оборачивает весь HTML-документ. Атрибут lang указывает язык страницы (например, "ru" для русского).

<head></head>

Голова документа - содержит метаинформацию о странице (заголовок, кодировку, ссылки на стили и скрипты), которая не отображается напрямую на странице.

<body></body>

Тело документа - содержит все видимое содержимое веб-страницы: текст, изображения, ссылки, таблицы, списки и т.д.

Тег <head> - что внутри?

Тег <head> содержит метаданные (данные о данных) HTML-документа. Эта информация не отображается на странице, но очень важна для браузеров и поисковых систем.

<head>
  <!-- Кодировка символов -->
  <meta charset="UTF-8">

  <!-- Настройки отображения на мобильных -->
  <meta name="viewport" content="width=device-width, initial-scale=1.0">

  <!-- Заголовок страницы (отображается во вкладке) -->
  <title>Мой сайт - Главная страница</title>

  <!-- Описание для поисковых систем -->
  <meta name="description" content="Это мой первый сайт на HTML и CSS">

  <!-- Подключение CSS файла -->
  <link rel="stylesheet" href="style.css">
</head>

Важные элементы внутри <head>:

  • <meta charset="UTF-8"> - определяет кодировку символов (поддержка кириллицы)
  • <meta name="viewport"> - делает сайт адаптивным для мобильных устройств
  • <title></title> - заголовок страницы (отображается во вкладке браузера)
  • <meta name="description"> - описание страницы для поисковых систем
  • <link rel="stylesheet"> - подключение CSS файлов
  • <script></script> - подключение JavaScript файлов

Тег <body> - видимое содержимое

Тег <body> содержит всё содержимое, которое отображается в окне браузера. Здесь размещаются все видимые элементы: текст, изображения, ссылки, формы и т.д.

Популярные теги внутри body:

<h1>-<h6>
<p>
<a>
<img>
<div>
<span>
<ul>, <ol>, <li>
<table>
<form>
<button>

Пример содержимого body:

<body>
  <header>
    <h1>Мой первый сайт</h1>
    <nav>
      <a href="index.html">Главная</a>
    </nav>
  </header>

  <main>
    <h2>Добро пожаловать!</h2>
    <p>Это мой первый сайт.</p>
  </main>

  <footer>
    <p>© 2023 Мой сайт</p>
  </footer>
</body>

Семантические теги HTML5

Семантические теги описывают смысл содержимого, а не только его внешний вид. Они улучшают доступность для скринридеров и SEO.

<header> Шапка сайта
<nav> Навигация
<main> Основное содержимое
<article> Статья
<aside> Боковая панель
<footer> Подвал сайта
Визуальное представление семантических тегов

<header>

Вводная часть раздела или страницы. Обычно содержит логотип, навигацию, заголовок.

<footer>

Нижняя часть раздела или страницы. Часто содержит информацию об авторских правах, контакты.

<nav>

Навигационные ссылки по сайту. Основное меню обычно помещается в этот тег.

<article>

Независимый, самодостаточный контент: статья в блоге, новость, пост на форуме.

<section>

Тематическая группировка контента. Обычно имеет собственный заголовок.

<aside>

Боковая панель, связанная с основным содержимым, но не являющаяся его частью.

HTML и CSS на практике

HTML создает структуру страницы, а CSS отвечает за её внешний вид. Вместе они формируют основу любого веб-сайта.

Практические примеры HTML

Создание формы обратной связи:

<form action="/submit" method="POST">
  <label for="name">Имя:</label>
  <input type="text" id="name" name="name" required>

  <label for="email">Email:</label>
  <input type="email" id="email" name="email" required>

  <label for="message">Сообщение:</label>
  <textarea id="message" name="message" rows="4"></textarea>

  <button type="submit">Отправить</button>
</form>

Создание таблицы:

<table>
  <thead>
    <tr>
      <th>Имя</th>
      <th>Возраст</th>
      <th>Город</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>Анна</td>
      <td>25</td>
      <td>Москва</td>
    </tr>
  </tbody>

Создание навигационного меню:

<nav>

  </ul>

Основы CSS (Cascading Style Sheets)

CSS используется для стилизации HTML-элементов. Стили можно задавать тремя способами:

Внешний CSS

Стили в отдельном файле с расширением .css, который подключается в HTML:

<link rel="stylesheet" href="style.css">

Внутренний CSS

Стили внутри тега <style> в секции <head>:

<style>
  body {
    font-family: Arial, sans-serif;
  }
</style>

Inline CSS

Стили непосредственно в атрибуте style HTML-элемента:

<h1 style="color: blue;">Заголовок</h1>

Пример CSS правил:

/* Селектор элемента */
h1 {
  color: #2c3e50;
  font-size: 2.5rem;
  text-align: center;
}

/* Селектор класса */
.card {
  background: white;
  border-radius: 8px;
  padding: 20px;
  box-shadow: 0 4px 10px rgba(0,0,0,0.1);
}

/* Селектор ID */
#header {
  background: linear-gradient(to right, #3498db, #2c3e50);
  color: white;
  padding: 40px 0;
}

/* Медиазапрос для адаптивности */
@media (max-width: 768px) {
  h1 {
    font-size: 1.8rem;
  }
}

Макеты и верстка в CSS

Современная верстка использует Flexbox и Grid для создания адаптивных макетов.

Flexbox пример:

/* CSS */
.container {
  display: flex;
  justify-content: space-between;
  align-items: center;
  flex-wrap: wrap;
}

.item {
  flex: 1 1 300px; /* flex-grow, flex-shrink, flex-basis */
  margin: 10px;
}

CSS Grid пример:

/* CSS */
.grid-container {
  display: grid;
  grid-template-columns: repeat(auto-fill, minmax(250px, 1fr));
  gap: 20px;
}

Совет: Используйте Flexbox для одномерных макетов (ряд или колонка), а Grid для двумерных макетов (и строки, и колонки).

JavaScript для веб-сайтов

JavaScript добавляет интерактивность на ваши веб-страницы. С его помощью можно создавать динамические элементы, обрабатывать данные и реагировать на действия пользователя.

Основы JavaScript

JavaScript — это язык программирования, который выполняется в браузере пользователя. Он позволяет делать сайты интерактивными.

Пример простого JavaScript кода:

// Переменные
let name = "Иван";
const age = 25;
var city = "Москва"; // устаревший способ

// Функция
function greet() {
  alert("Привет, " + name + "!");
}

// Условия
if (age >= 18) {
  console.log("Совершеннолетний");
} else {
  console.log("Несовершеннолетний");
}

// Цикл
for (let i = 0; i < 5; i++) {
  console.log("Итерация: " + i);
}

Типы данных в JavaScript:

String (строка)
Number (число)
Boolean (логический)
Array (массив)
Object (объект)
Null
Undefined

Работа с DOM (Document Object Model)

DOM — это представление HTML-документа в виде дерева объектов. JavaScript может изменять DOM, чтобы динамически обновлять содержимое страницы.

Основные методы работы с DOM:

// Получение элементов
const elementById = document.getElementById('myId');
const elementsByClass = document.getElementsByClassName('myClass');
const elementsByTag = document.getElementsByTagName('div');
const elementQuery = document.querySelector('.myClass');
const elementsQueryAll = document.querySelectorAll('div.myClass');

// Изменение содержимого
elementById.innerHTML = 'Новый текст';
elementById.textContent = 'Текст без HTML';

// Изменение стилей
elementById.style.color = 'red';
elementById.classList.add('new-class');
elementById.classList.remove('old-class');

// Создание новых элементов
const newElement = document.createElement('div');
newElement.textContent = 'Я новый элемент';
document.body.appendChild(newElement);

Обработка событий:

// Способ 1: через атрибут
<button onclick="handleClick()">Нажми меня</button>

// Способ 2: через свойство
buttonElement.onclick = function() {
  alert('Кнопка нажата!');
};

// Способ 3: через addEventListener (рекомендуется)
buttonElement.addEventListener('click', function() {
  console.log('Кнопка нажата!');
});

// HTML
<div class="tabs">
  <button class="tab active" data-tab="tab1">Вкладка 1</button>
  <button class="tab" data-tab="tab2">Вкладка 2</button>
</div>

<div class="tab-content active" id="tab1">Содержимое 1</div>
<div class="tab-content" id="tab2">Содержимое 2</div>

// JavaScript
document.addEventListener('DOMContentLoaded', function() {
  const tabs = document.querySelectorAll('.tab');
  const contents = document.querySelectorAll('.tab-content');

  tabs.forEach(tab => {
    tab.addEventListener('click', () => {
      // Убираем активный класс у всех
      tabs.forEach(t => t.classList.remove('active'));
      contents.forEach(c => c.classList.remove('active'));

      // Добавляем активный класс текущей вкладке
      tab.classList.add('active');
      const tabId = tab.getAttribute('data-tab');
      document.getElementById(tabId).classList.add('active');
    });
  });
});

Дизайн веб-сайтов

Хороший дизайн делает сайт не только красивым, но и удобным для пользователя. В этом разделе рассмотрим основные принципы веб-дизайна.

Принципы хорошего веб-дизайна

1 Простота

Минималистичный дизайн помогает пользователям сфокусироваться на важном контенте. Избегайте лишних элементов и визуального шума.

Правило: Если элемент не выполняет функцию, уберите его.

2 Навигация

Меню должно быть интуитивно понятным и доступным с любой страницы. Пользователь всегда должен понимать, где он находится.

Правило "трех кликов": Любая информация должна быть доступна за 3 клика.

3 Адаптивность

Сайт должен хорошо выглядеть на всех устройствах: компьютерах, планшетах и смартфонах. Используйте медиазапросы CSS.

Подход: Mobile First - сначала дизайн для мобильных, потом для десктопов.

4 Скорость загрузки

Пользователи уходят с сайтов, которые загружаются дольше 3 секунд. Оптимизируйте изображения и код.

Сжатие изображений
Минификация кода
Кэширование

5 Доступность

Сайт должен быть доступен людям с ограниченными возможностями. Используйте семантические теги, alt для изображений, достаточный контраст.

Стандарт: WCAG (Web Content Accessibility Guidelines)

6 Консистентность

Единый стиль на всех страницах: одинаковые цвета, шрифты, кнопки, отступы. Это создает профессиональное впечатление.

Используйте: CSS-переменные для единообразия.

Цветовая палитра

Правильный выбор цветов влияет на восприятие сайта. Используйте контрастные цвета для текста и фона. Ограничьте палитру 3-4 основными цветами.

Типографика

Типографика — это искусство оформления текста. Правильная типографика улучшает читаемость и восприятие контента.

Основные правила типографики:

  • Используйте не более 2-3 шрифтов на сайте - один для заголовков, один для основного текста, третий (опционально) для акцентов.
  • Обеспечьте хорошую читаемость текста - достаточный контраст между текстом и фоном.
  • Используйте размер шрифта не менее 16px для основного текста - для комфортного чтения на всех устройствах.
  • Применяйте иерархию заголовков - h1, h2, h3, h4 должны отличаться по размеру и весу.
  • Используйте межстрочный интервал 1.5-1.6 - для лучшей читаемости длинных текстов.
  • Ограничьте длину строки 50-75 символов - чтобы глаза не уставали при чтении.

Публикация сайта

После того как вы создали сайт, его нужно разместить в интернете, чтобы его могли увидеть другие пользователи. Этот процесс называется деплоем (развертыванием).

Публикация сайта на GitHub Pages

GitHub Pages — это бесплатный хостинг от GitHub для статических сайтов. Он идеально подходит для личных сайтов, документации и проектов.

GitHub Pages - пошаговая инструкция

1

Создайте аккаунт на GitHub

Если у вас ещё нет аккаунта, зарегистрируйтесь на github.com.

2

Создайте новый репозиторий

Назовите его username.github.io, где username — ваш логин на GitHub.

3

Загрузите файлы вашего сайта

Загрузите HTML, CSS, JS файлы и другие ресурсы в репозиторий. Главный HTML-файл должен называться index.html.

4

Включите GitHub Pages

В настройках репозитория найдите раздел "Pages" и выберите ветку main как источник.

5

Дождитесь публикации

GitHub автоматически опубликует ваш сайт. Обычно это занимает 1-2 минуты. Ваш сайт будет доступен по адресу https://username.github.io.

Преимущества GitHub Pages: бесплатно, простота использования, интеграция с Git, поддержка кастомных доменов, SSL-сертификат включен.

Публикация сайта на Netlify

Netlify — это мощная платформа для развертывания веб-проектов с поддержкой непрерывного развертывания, SSL, форм и функций сервера.

Netlify - пошаговая инструкция

1

Создайте аккаунт на Netlify

Зарегистрируйтесь на netlify.com используя email, GitHub, GitLab или Bitbucket.

2

Загрузите файлы сайта

На главной странице Netlify перетащите папку с файлами вашего сайта в область "Drag and drop your site folder here".

3

Или подключите Git-репозиторий

Нажмите "New site from Git", выберите ваш Git-провайдер (GitHub, GitLab, Bitbucket) и репозиторий с сайтом.

4

Настройте сборку (опционально)

Если ваш сайт использует сборщики (как React, Vue), укажите команды сборки. Для простых HTML/CSS сайтов настройки не требуются.

5

Сайт опубликован!

Netlify автоматически присвоит вашему сайту уникальный URL (например, https://amazing-site-12345.netlify.app). Вы можете настроить кастомный домен.

Преимущества Netlify: автоматическое SSL, непрерывное развертывание, формы, функции сервера, разделение на окружения (production/preview), поддержка редиректов.

Публикация сайта на Vercel

Vercel — это платформа для фронтенд-разработчиков, созданная командой Next.js. Она отлично подходит для современных веб-приложений и статических сайтов.

Vercel - пошаговая инструкция

1

Создайте аккаунт на Vercel

Зарегистрируйтесь на vercel.com используя GitHub, GitLab или Bitbucket.

2

Импортируйте Git-репозиторий

Нажмите "Add New" → "Project", выберите Git-провайдер и репозиторий с вашим сайтом.

3

Настройте проект

Для статических HTML/CSS сайтов Vercel определит настройки автоматически. Если нужно, измените настройки сборки.

4

Дождитесь деплоя

Vercel автоматически развернет ваш сайт. Процесс обычно занимает меньше минуты.

5

Настройте кастомный домен

В настройках проекта добавьте кастомный домен. Vercel автоматически настроит SSL-сертификат.

Преимущества Vercel: отличная производительность, автоматическое SSL, интеграция с Next.js, серверные функции, предварительный просмотр для pull requests.

Рекомендация для начинающих: начните с GitHub Pages, так как это самый простой вариант. Когда освоитесь, попробуйте Netlify или Vercel для более продвинутых функций.

Общие шаги для публикации любого сайта:

  1. Выберите хостинг-провайдера - компанию, которая будет хранить файлы вашего сайта на своих серверах.
  2. Купите доменное имя - адрес вашего сайта в интернете (например, mywebsite.com).
  3. Загрузите файлы на сервер - используйте FTP-клиент или встроенные инструменты хостинга.
  4. Настройте DNS - свяжите доменное имя с сервером хостинга.
  5. Протестируйте сайт - убедитесь, что все работает корректно на разных устройствах.
